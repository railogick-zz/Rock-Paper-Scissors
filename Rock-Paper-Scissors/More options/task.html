Description

How about some brand new rules? The original game has a fairly small choice of options.

Extended rules are decreasing the probability of drawing, so it should be really cool to have it.
Now, your program should accept alternative lists of options, like this:

 

Also, there is a very popular version of this game called Rock, Paper, Scissors, Lizard, Spock.

Let's create a more abstract version of the game, where the user first chooses the options that will be used in the game. Thus, the user will be able to play in the original version, and in the version of 5 options, and in the version of 15 options. Note that the number of options must be odd, otherwise you will not be able to create honest rules. One option should be worse than half of the options, and better than the other half of the options, and equal to itself.

After entering the name, the user should provide a list of options separated by a comma. 

If user lefts options unfilled, use rock,paper,scissors.
